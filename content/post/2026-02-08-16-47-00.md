<p style="">OpenClaw 是一个你可以部署在自己设备上的个人 AI 助理：它通过你常用的聊天软件（如 Telegram/WhatsApp/Slack/Discord 等）和你对话，也能运行工具（浏览器、文件、定时任务、节点设备等），核心由一个常驻后台的 Gateway 负责调度。</p><p style="">本文以“最快跑起来 + 最少踩坑”为目标，把官方文档与社区教程的关键步骤整理成一篇可直接照做的安装指南。</p><h2 style="" id="1.-%E5%AE%89%E8%A3%85%E5%89%8D%E5%87%86%E5%A4%87">1. 安装前准备</h2><h3 style="" id="1.1-%E7%B3%BB%E7%BB%9F%E4%B8%8E%E8%BF%90%E8%A1%8C%E6%97%B6">1.1 系统与运行时</h3><ul><li><p style="">Node.js：建议 <strong>Node &gt;= 22</strong>（官方文档以此为前提）</p></li><li><p style="">macOS / Linux：直接按本文步骤即可</p></li><li><p style="">Windows：建议使用 <strong>WSL2 (Ubuntu)</strong>，原生 Windows 环境兼容性与工具链较麻烦</p></li></ul><p style="">验证 Node 版本：</p><p style="">“`bash</p><p style="">node -v</p><p style="">“`</p><h2 style="" id="2.-%E5%AE%89%E8%A3%85-openclaw-cli">2. 安装 OpenClaw CLI</h2><p style="">官方推荐一键脚本（macOS / Linux）：</p><p style="">“`bash</p><p style="">curl -fsSL https://openclaw.ai/install.sh | bash</p><p style="">“`</p><p style="">Windows PowerShell：</p><p style="">“`powershell</p><p style="">iwr -useb https://openclaw.ai/install.ps1 | iex</p><p style="">“`</p><p style="">也可以选择全局安装：</p><p style="">“`bash</p><p style="">npm install -g openclaw@latest</p><h1 style="" id="%E6%88%96">或</h1><p style="">pnpm add -g openclaw@latest</p><p style="">“`</p><h2 style="" id="3.-%E8%BF%90%E8%A1%8C%E5%BC%95%E5%AF%BC%E5%90%91%E5%AF%BC%EF%BC%88onboard%EF%BC%89">3. 运行引导向导（onboard）</h2><p style="">安装完成后，建议直接跑向导，它会把“模型、网关、工作区、渠道、技能、后台服务”等一次性配好：</p><p style="">“`bash</p><p style="">openclaw onboard –install-daemon</p><p style="">“`</p><p style="">向导会让你选择：</p><ul><li><p style="">Gateway 运行在本机还是远程</p></li><li><p style="">选择模型与鉴权方式（OAuth / API Key 等）</p></li><li><p style="">是否配置渠道（Telegram/WhatsApp/Discord…）</p></li><li><p style="">是否安装为后台服务（systemd/launchd；WSL2 可用 systemd）</p></li></ul><p style="">提示：如果你要用 Telegram/WhatsApp，官方不建议用 Bun 作为运行时，优先用 Node（兼容性更稳）。</p><h2 style="" id="4.-%E5%90%AF%E5%8A%A8%E4%B8%8E%E8%AE%BF%E9%97%AE-gateway%EF%BC%88%E6%9C%80%E5%85%B3%E9%94%AE%EF%BC%89">4. 启动与访问 Gateway（最关键）</h2><p style="">如果你在向导里选择安装后台服务，通常 Gateway 已经在跑了：</p><p style="">“`bash</p><p style="">openclaw gateway status</p><p style="">“`</p><p style="">如果你要手动前台启动（便于看日志排障）：</p><p style="">“`bash</p><p style="">openclaw gateway –port 18789 –verbose</p><p style="">“`</p><p style="">然后打开 Dashboard / Control UI：</p><ul><li><p style="">默认地址<code>http://127.0.0.1:18789/</code></p></li><li><p style="">或直接用命令：</p></li></ul><p style="">“`bash</p><p style="">openclaw dashboard</p><p style="">“`</p><h2 style="" id="5.-2-%E5%88%86%E9%92%9F%E8%87%AA%E6%A3%80%EF%BC%88%E6%8E%A8%E8%8D%90%EF%BC%89">5. 2 分钟自检（推荐）</h2><p style="">当你不确定“到底哪里坏了”，先跑这些：</p><p style="">“`bash</p><p style="">openclaw status</p><p style="">openclaw health</p><p style="">openclaw security audit –deep</p><p style="">“`</p><ul><li><p style=""><code>status</code>：运行状态与基础信息</p></li><li><p style=""><code>health</code>：健康检查（常见提示：未配置模型鉴权等）</p></li><li><p style=""><code>security audit</code>：安全审计（尤其是 DM 策略、允许列表等）</p></li></ul><h2 style="" id="6.-%E8%BF%9E%E6%8E%A5-telegram%EF%BC%88%E6%9C%80%E5%B8%B8%E7%94%A8%EF%BC%89">6. 连接 Telegram（最常用）</h2><p style="">两件事：</p><p style="">1) 在 Telegram 找到 BotFather 创建机器人，拿到 bot token</p><p style="">2) 在 OpenClaw 里配置 Telegram channel（向导能做；你也可以手动配置）</p><p style="">重要的默认安全策略：</p><ul><li><p style=""><strong>陌生人私聊默认是 pairing 模式</strong>：对方发来第一条消息时会收到一个配对码，但消息不会被处理</p></li><li><p style="">你需要批准配对，机器人才会开始正常回复</p></li></ul><p style="">查看与批准配对示例：</p><p style="">“`bash</p><p style="">openclaw pairing list telegram</p><p style="">openclaw pairing approve telegram</p><p style=""><code>```</code></p><p style=""><code>如果你“发消息机器人不回”，十有八九是：</code></p><ul><li><p style=""><code>Gateway 没启动</code></p></li><li><p style=""><code>Telegram bot token 配错</code></p></li><li><p style=""><code>还没 approve pairing</code></p></li></ul><h2 style="" id="7.-%E5%B8%B8%E8%A7%81%E9%97%AE%E9%A2%98%EF%BC%88%E8%B8%A9%E5%9D%91%E9%80%9F%E6%9F%A5%EF%BC%89"><code>7. 常见问题（踩坑速查）</code></h2><h3 style="" id="7.1-openclaw%3A-command-not-found"><code>7.1 openclaw: command not found</code></h3><p style=""><code>通常是 npm 全局 bin 路径没加入 PATH：</code></p><p style=""><code>```bash</code></p><p style=""><code>npm prefix -g</code></p><p style=""><code>```</code></p><p style=""><code>把 $(npm prefix -g)/bin 加到 PATH（例如写入 ~/.zshrc / ~/.bashrc），然后重开终端。</code></p><h3 style="" id="7.2-node-%E7%89%88%E6%9C%AC%E5%A4%AA%E4%BD%8E-%2F-%E7%8E%AF%E5%A2%83%E6%B7%B7%E4%B9%B1"><code>7.2 Node 版本太低 / 环境混乱</code></h3><ul><li><p style=""><code>用 node -v 确认 &gt;= 22</code></p></li><li><p style=""><code>Windows 请优先 WSL2</code></p></li></ul><h3 style="" id="7.3-dashboard-%E6%89%93%E4%B8%8D%E5%BC%80"><code>7.3 Dashboard 打不开</code></h3><ul><li><p style=""><code>先确认 openclaw gateway status 显示 running</code></p></li><li><p style=""><code>确认端口未冲突（默认 18789）</code></p></li><li><p style=""><code>本机访问用 127.0.0.1，远程访问需要做安全加固（token / tunnel / VPN）</code></p></li></ul><h2 style="" id="8.-%E4%B8%8B%E4%B8%80%E6%AD%A5%E6%80%8E%E4%B9%88%E7%8E%A9%EF%BC%88%E5%BB%BA%E8%AE%AE%E8%B7%AF%E7%BA%BF%EF%BC%89"><code>8. 下一步怎么玩（建议路线）</code></h2><ul><li><p style=""><code>先在 Dashboard
